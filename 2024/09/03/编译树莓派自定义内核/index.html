编译树莓派自定义内核

发表于 2024-09-03 更新于 2025-12-09 分类于 技术 ， 树莓派
  • 新的项目属于国外引进项目。运行在树莓派上面。需要更换开机界面放上自己的 logo 什么的。在开机的时候屏幕上会有一个光标一直在闪。想把那个光标给去掉，查阅资料发现需要重新编译树莓派的内核。在此做下记录。

本文介绍如何在树莓派上编译自定义内核，包括工具下载、内核配置和编译过程。

1. 工具下载

sudo apt-get install -y bc build-essential libncurses5-dev libssl-dev
从 github 上下载 raspberry Linux 内核源码
git clone --depth=1 -b rpi-4.8.y https://github.com/raspberrypi/linux.git

2. 配置内核

因为我们是在树莓派上编译，直接使用当前树莓派自带的编译器就可以，可以使用
arm-linux- 加 Tab 键进行查看，我的树莓派上的交叉编译器为 arm-linux-gnueabihf-

    1. 切换到你下载的内核目录下 cd ./linux-rpi-4.19.y
      在配置内核的时候，需要先配置 config。这里多说一句网上需要博客里面记载的配置 config 的命令是
      make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- bcmrpi3_defconfig 这个是配置 64 位的并且指定编译器。我们在树莓派上编译的，查看当前系统 getconf LONG_BIT, 如果返回结果是 32 则系统是 32 位的，如果返回的是 64 则系统是 64 位的。系统里面自带的就是交叉编译器。所以只要找到配置文件就可以，使用 find ./ -name "*bcm*defconfig*" 查找下对应的默认配置，只保留arm下的结果 pi 1 使用的事 bcmrpi_defconfig pi 2 3 使用的是 bcm 2709_defconfig，make bcm2709_defcongfig。
    1. 因为我需要配置树莓派开机界面进行配置，所以 make menuconfig。会出现内核配置界面

      我把终端输出给去掉了。具体修改为 Device Drivers->Character Devices->[]support for console on virtual terminal

3 创建新的内核

在目录下直接 make -j4 开始编译内核，晚上下班后我开始编译的，然后树莓派一直开着就没关，具体多长时间没统计。
编译完成后去./linux-rpi-4.14. y/arch/arm/boot 下找到自己编译完成的内核

内核编译完成后在往内存卡上拷贝上发现内存卡上内存不足。无法进行拷贝。后移植到 ubuntu 下进行编译

4 在 ubuntu 下编译

在 ubuntu 下编译需要需要下载树莓派官方工具。树莓派官方工具
下载官方工具因为我们要使用官方提供的 arm-linux-gcc 工具

    1. 将下载好的树莓派内核拷贝到 ubuntu 下，个人建议新建一个文件夹。我创建的是 raspberryKernel。我们后续操作都是在这个目录下。
    1. 修改工具链。网上大部分的办法都是配置环境变量。因为我 ubuntu 有一个交叉编译工具。不想重新配置了。所以直接去修改 Makefile 文件 vim Makefile, 找到 ARCH 第一次出现的地方。添加交叉工具链

      这种方式以后我们在 make 的时候不需要每次都去指定编译器。
    1. 剩下的步骤和在树莓派上一样。这里不在具体说明。

5 移植自己的内核

    1. 在树莓派官网下载. img 镜像。拷贝到 ubuntu 上面。然后将这个镜像挂载到自己的 ubuntu 上。具体方法如下
      fdisk -l xxxxxx.img xxx 是你自己下载的镜像。
    1. 配置内核模块
    1. 替换自己的内核
    1. 卸载挂载的内核
      umount /mnt/boot
      umount /mnt
      将自己编译的内核替换原先的内核。然后将 xxxxxxxx. img 你自己下载的镜像重新烧写到内存卡中就可以了。
